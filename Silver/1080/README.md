# boj_1080_matrix

### 📌 문제 정보
- **난이도**: Silver 1
- **유형**: 그리디 알고리즘 (Greedy), 시뮬레이션
- **문제 링크**: [백준 1080번](https://www.acmicpc.net/problem/1080)

### 💡 풀이 아이디어 (수학적 원리)
- **그리디의 정당성**: 특정 칸 $(i, j)$를 포함하는 $3 \times 3$ 연산 중, $(i, j)$가 왼쪽 위 꼭짓점이 되는 연산은 단 하나뿐이다. 따라서 $(0, 0)$부터 순차적으로 탐색하며 $A[i][j]$와 $B[i][j]$가 다를 경우 즉시 뒤집기를 수행하면, 이미 지나온 칸은 다시 고려할 필요가 없다.
- **연산의 독립성**: 같은 위치를 두 번 뒤집으면 원래 상태로 돌아오므로, 각 위치에서의 연산은 최대 1번만 수행한다.
- **최종 검증**: $3 \times 3$ 연산을 수행할 수 있는 최대 범위($(N-3, M-3)$)까지 모두 처리한 후, 결과 행렬 $A$가 $B$와 완벽히 일치하는지 확인하여 불가능한 경우(-1)를 가려낸다.

### 🛠️ 개발 환경 및 트러블슈팅
- **Language**: Python 3.14
- **IDE**: VS Code (Interactive Window & Terminal 활용)
- **Trouble Shooting**: 
    - **입력 데이터 구조**: C언어의 고정 배열과 달리 파이썬에서는 `list(map(int, input()))`를 활용해 문자열을 정수 리스트로 변환하고, 이를 다시 중첩 리스트(`[[...]]`)로 쌓아 2차원 행렬을 동적으로 생성함.
    - **Index Error 방지**: $3 \times 3$ 연산 시 인덱스가 범위를 벗어나지 않도록 루프 범위를 N-2, M-2로 제한함.
    - **예외 케이스 처리**: 행렬의 크기가 $3 \times 3$보다 작을 때 무조건 `-1`을 출력하는 것이 아니라, 연산이 불가능하더라도 이미 두 행렬이 같다면 `0`을 출력해야 한다는 점을 간과하여 오답 발생. 마지막에 `A == B` 조건을 통해 최종 정답을 도출하도록 수정함.
